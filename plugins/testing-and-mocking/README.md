# testing-and-mocking

TypeScript・React プロジェクトのテストと MSW モックのベストプラクティスを提供するプラグイン。

## 概要

テスト設計・アサーション方針・MSW ハンドラーパターン・モックデータ規約を skill として提供する。テストファイルや MSW 関連ディレクトリを開いているときに自動ロードされる。

## 提供コンポーネント

### Skills

| スキル名 | 説明 |
|----------|------|
| `testing-and-mocking` | テストと MSW モックのベストプラクティスガイド |

**スキルが自動ロードされるファイルパターン**:
- `**/*.test.ts` / `**/*.test.tsx`
- `**/*.spec.ts` / `**/*.spec.tsx`
- `**/msw/**`
- `**/mocks/**`
- `**/__mocks__/**`

## カバーする領域

| 領域 | 内容 |
|------|------|
| テスト設計 | コンポーネントのロジック export・パターン変更時のテスト更新・多ファイル変更後の追従確認 |
| アサーション | `toEqual` vs `toMatchObject` の使い分け・non-null assertion 回避 |
| MSW ハンドラー | ハンドラー追加の必須化・フィルター対象の一致・`withAuth` シグネチャ |
| モックデータ | RFC 2606 準拠の `example.com` 使用・環境判定のユーティリティ化 |

## フック

なし
