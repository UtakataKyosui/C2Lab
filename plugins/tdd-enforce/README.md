# tdd-enforce

テスト駆動開発（TDD）を実践・強制するプラグイン。

## 概要

Red-Green-Refactor サイクルの実践ガイドと TDD 準拠チェックを提供する。実装ファイルの保存前にテストファイルの存在を確認し、TDD ワークフローを促進する。

## 提供コンポーネント

### Commands (Skills)

| コマンド | 説明 |
|----------|------|
| `init` | プロジェクトの TDD 環境を初期化する。言語検出・テストフレームワーク提案・テスト設定の確認を行う。 |
| `cycle` | TDD の Red-Green-Refactor サイクルをフルガイドモードで実行する。各ステップでユーザー確認を取りながら進める。 |
| `suggest-framework` | プロジェクトの言語・フレームワークに基づいてテストフレームワークを提案する |

### Agents

| エージェント | 説明 |
|--------------|------|
| `tdd-compliance-checker` | プロジェクト内の実装コードに対応するテストが存在するか検証し、TDD 準拠状況をレポートする |

### Skills

| スキル名 | 説明 |
|----------|------|
| `tdd-enforce` | TDD の方法論と Red-Green-Refactor サイクルの実践ガイド。テストの書き方・フレームワーク選定・TDD パターンを提供する。 |

**スキルが自動ロードされるファイルパターン**:
- `**/*.test.*`
- `**/*.spec.*`
- `**/*_test.*`
- `**/test_*.*`
- `**/tests/**`
- `**/__tests__/**`

### ガイド

| ファイル | 内容 |
|----------|------|
| `tdd-patterns.md` | TDD パターン集（Given-When-Then, AAA 等） |
| `testing-frameworks.md` | 言語別テストフレームワーク比較・選定ガイド |
| `test-design.md` | テスト設計の原則（単体・統合・E2E の分類等） |

## 使い方

```
/init                    # TDD 環境をセットアップ
/cycle                   # Red-Green-Refactor サイクルを開始
/suggest-framework       # プロジェクトに適したフレームワークを提案
```

テストファイルを開いているとき、`tdd-enforce` スキルが自動的に参照される。

## フック

| イベント | 対象 | 処理 |
|----------|------|------|
| `PreToolUse` | Edit / Write | 実装ファイルを変更する前に `check-test-exists.py` で対応するテストファイルの存在を確認する |
