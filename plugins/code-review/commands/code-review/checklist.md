# コードレビュー・チェックリスト

## 概要
品質・可読性・保守性・セキュリティを確保するための徹底的なコードレビュー用チェックリスト。プログラムを書く上で最も心がけるべきは、安全性や速度だけでなく、「見やすいコード」である

## レビューカテゴリ（ベースライン）

### 機能・ロジック
- [ ] コードが意図どおりに動作している
- [ ] 例外的なケースまで考慮されている
- [ ] エラーハンドリングが適切で、エラーメッセージが明確になっている
- [ ] **処理フローが論理的かつ自然な順序で配置されている**
- [ ] **ネストが深すぎず、ガード節が適切に活用されている**
- [ ] **入力引数を変更する前にコピーし、予期せぬ副作用を防いでいる**
- [ ] 明らかなバグやロジックエラーがない

### コード品質・保守性
- [ ] **コードの構造がシンプルで、冗長な部分や不要なコードが排除されている**
- [ ] **関数/クラスが単一責任の原則に従い、論理的な単位に分割されている**
- [ ] **変数名/関数名が具体的かつ簡潔で、役割を正確に表している（英単語の使用）**
- [ ] **命名規則/スタイル（インデント、カッコなど）が一貫している**
- [ ] 再利用可能な関数やクラスが作成されている（重複の排除）
- [ ] **マジックナンバーを使用せず、意味のある定数や変数に置き換えられている**
- [ ] **意図や複雑な処理に適切なコメントが追加され、コードから自明なコメントは削除されている**
- [ ] プロジェクトの規約に従っている

### パフォーマンス
- [ ] **効率的なアルゴリズムとデータ構造が選択されている**
- [ ] **ループの最適化や不要な処理の削減がされている**
- [ ] メモリ管理が適切に行われている

### セキュリティ・システム安定性
- [ ] 明らかなセキュリティ上の脆弱性がない
- [ ] 入力バリデーションが実装されている
- [ ] 機微なデータが適切に扱われている
- [ ] シークレットがハードコードされていない
- [ ] **外部APIへの依存処理が特定箇所に制限されている**
