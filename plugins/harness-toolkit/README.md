# harness-toolkit

Modern CLI harness for Claude Code — integrates Jujutsu VCS, Rust-based code exploration, AI context generation, security scanning, and dependency management into your development workflow.

## Overview

`harness-toolkit` provides Claude Code with deep knowledge and automation for a curated set of modern CLI tools:

| Category | Tools |
|---|---|
| VCS | [Jujutsu (jj)](https://github.com/jj-vcs/jj) |
| Code Exploration | [fd](https://github.com/sharkdp/fd), [ripgrep](https://github.com/BurntSushi/ripgrep), [tre](https://github.com/dduan/tre) |
| Context Generation | [repomix](https://github.com/yamadashy/repomix), [rtk](https://github.com/rtk-ai/rtk) |
| Dependency Management | [proto](https://github.com/moonrepo/proto), [Knip](https://github.com/webpro-nl/knip), [dependency-cruiser](https://github.com/sverweij/dependency-cruiser), [ncu](https://github.com/raineorshine/npm-check-updates) |
| Package Security | [Takumi Guard](https://github.com/flatt-security/setup-takumi-guard-npm) |
| App Security | [Semgrep](https://github.com/semgrep/semgrep), [type-coverage](https://github.com/plantain-00/type-coverage), [ghasec](https://github.com/koki-develop/ghasec) |
| Parallel Execution | [rust-parallel](https://github.com/aaronriekenberg/rust-parallel) |

## Features

### Skills (Knowledge Base)

7 auto-activating skills that Claude loads when relevant:

- **`vcs-jj`** — Jujutsu VCS commands, workspace management, and workflows
- **`code-exploration`** — fd/ripgrep/tre patterns for fast file and code search
- **`context-generation`** — repomix compression and rtk token reduction
- **`dependency-management`** — proto/Knip/dependency-cruiser/ncu workflows
- **`package-security`** — Takumi Guard setup and supply chain attack prevention
- **`app-security`** — Semgrep/type-coverage/ghasec scanning and remediation
- **`parallel-execution`** — rust-parallel patterns for concurrent command execution

### Hooks (Automation)

4 event-driven hooks that run automatically:

| Hook | Event | Action |
|---|---|---|
| `check-tools` | SessionStart | Checks for missing tools, suggests harness-setup |
| `security-scan` | PostToolUse (Write/Edit) | Runs Semgrep on the modified file |
| `repomix-context` | Stop | Generates `CONTEXT.md` from changed files (diff-based) |
| `repomix-full` | PreCompact | Saves full codebase snapshot before context compaction |

### Agents

- **`security-reporter`** — Runs Semgrep/ghasec/type-coverage and generates a structured security report
- **`harness-setup`** — Detects missing tools and provides platform-specific installation commands

## Installation

### From Marketplace

```bash
# Install via Claude Code CLI
cc --install-plugin harness-toolkit
```

### Local Development

```bash
git clone https://github.com/UtakataKyosui/C2Lab
cc --plugin-dir C2Lab/plugins/harness-toolkit
```

## Prerequisites

### Required Tools

Install all required tools before enabling the plugin:

**macOS (Homebrew + npm + cargo):**
```bash
# Rust CLI tools
brew install fd ripgrep

# Jujutsu VCS
brew install jj

# AI context tools
npm install -g repomix

# Security
brew install semgrep
```

**Linux:**
```bash
sudo apt install fd-find ripgrep
cargo install jujutsu
npm install -g repomix
pip install semgrep
```

### Optional Tools

```bash
cargo install rtk                           # rtk (token reduction)
brew install tre-command                    # tre (directory tree)
cargo install rust-parallel                 # rust-parallel
curl -fsSL https://moonrepo.dev/install/proto.sh | bash  # proto
npm install -g knip                         # Knip
go install github.com/koki-develop/ghasec@latest  # ghasec
npx npm-check-updates --version &>/dev/null || npm install -g npm-check-updates  # ncu
```

## Usage

### Skills

Skills activate automatically when you ask Claude about relevant topics:

```
# Activates vcs-jj skill
"jj でブランチを作りたい"
"Jujutsu のワークスペースを使って並行開発したい"

# Activates code-exploration skill
"fd で TypeScript ファイルを検索したい"
"ripgrep でコード内のパターンを検索して"

# Activates context-generation skill
"repomix でコードベースを圧縮してコンテキストを生成して"
"rtk でトークンを削減しながら出力したい"

# Activates app-security skill
"Semgrep でセキュリティスキャンしたい"
"GitHub Actions のセキュリティ設定を確認して"
```

### Agents

```
# Trigger security-reporter
"セキュリティレポートを生成して"
"Semgrep と ghasec でスキャンして結果をまとめて"

# Trigger harness-setup (or automatically on SessionStart if tools are missing)
"harness-toolkit のセットアップをしたい"
"不足ツールをインストールしたい"
```

### Generated Files

The plugin generates these files in your project root:

| File | Generated by | Purpose |
|---|---|---|
| `CONTEXT.md` | Stop hook (repomix-context) | Diff-based context for next session |
| `CONTEXT_FULL.md` | PreCompact hook (repomix-full) | Full snapshot before compaction |

Add these to `.gitignore` if needed:
```
CONTEXT.md
CONTEXT_FULL.md
```

## Hook Configuration

All hooks gracefully skip when their required tools are not installed. No configuration is needed.

To disable a specific hook, remove its entry from `hooks/hooks.json` or override in your project's `.claude/settings.json`.
